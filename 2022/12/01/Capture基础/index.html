库介绍

库路径

cadence / SPB_17.4 / tools / capture / library

文件后缀： * .olb

库名称

  1. AMPLIFIER. 0LB共182个零件，存放模拟放大器IC，如CA3280， TL027C， EL4093;
  2. ARITHMETIC. 0LB共182个零件，存放逻辑运算IC，如TC4032B， 74LS85等 ;
  3. ATOD. 0LB共618个零件，存放A/D转换IC，如ADC0804， TC7109等 ;
  4. BUS DRIVERTRANSCEIVER. 0LB共632个零件，存放汇流排驱动IC，如74LS244， 74LS373 .等数字IC;
  5. ==CAPSYM. 0LB共35个零件，存放电源，地，输入输出口，标题栏等;==
  6. ==CONNECTOR. OLB共816个零件，存放连接器，如4 HEADER, CON AT62，RCA JACK;==
  7. COUNTER. 0LB共182个零件， 存放计数器IC,如74LS90， CD4040B;
  8. ==DISCRETE. 0LB共872个零件， 存放分立式元件，如电阻，电容，电感，开关，变压器等常用零件;==
  9. DRAM. OLB共623个零件，存放动态存储器，如TMS44C256， MN- 10等;
  10. ELECTRO MECHANICAL. 0LB共6个零件， 存放马达，断路器等电机类元件;
  11. FIF0.0LB共177个零件， 存放先进先出资料暂存器，如，SN74LS232;
  12. FILTRE. 0LB共80个零件， 存放滤波器类元件，如MAX270， LTC1065等;
  13. FPGA. 0LB存 放可编程逻辑器件，如XC6216/LCC;
  14. GATE. 0LB共691个零件， 存放逻辑门(含CMOS和TLL) ;
  15. LATCH. 0LB共305个零件， 存放锁存器，如4013， 74LS73， 74LS76等;
  16. LINE DRIVER RECEIVER. 0LB共380个零件， 存放线控驱动与接收器。如SN, DS275;
  17. MECHANICAL. 0LB共110个零件， 存放机构图件，如M HOLE 2，PGASOC-15-F等;
  18. MICROCONTROLLER. 0LB共523个零件， 存放单晶片微处理器，如68HC11， AT89C51;
  19. MICRO PROCESSOR. 0LB共288个零件， 存放微处理器;
  20. MISC. 0LB共1567个零件， 存放杂项图件，如电表(METER MA)，微处理器周边(Z80-DMA)等未分类的零件;

元件库创建办法

  1. File —> new —> Library (save as)此时为库名
  2. 改文件名 （*.olb）
  3. 放置矩形和pin ，批量为 Place Pin Array （正负位序可调）
  4. Eidt Pin 统一修改Pin名称 和pin type属性（Power , 常规为 Passive ）
  5. 修改元件名后保存 【路径：Design Resources / Library / 【自定义路径库】/ 元件名】

库的管理

1、通过新建元器件库，保存在本地，积少很多，作为常用的本地库
2、添加平时网上收集的库以及系统自带的库，作为常用的本地库
3、通过原理图来获取元器件库，作为常用的本地库

电源、地联通

1、系统自带电源/地连接符的库，不用添加，直接点击放置电源/地的按钮放置;
2、电源/地是全局变量，不同的页面也可以进行连接，不用添加分页符;
3、自定义电源/地连接符的Name名称，即可将原理图上电源/地全部联通。

添加BUS总线

1、总线代表的是具有相同电气特性的一-组导线，在具有相同电气特性的导线数目较多的情况下，可采用总线的方式;
2、点击键盘的B键放置总线，或者使用菜单栏中的Place- Bus功能,添加总线;
3、总线命名与网络标号的命名命令类似，通过点击N键或者或者点击菜单栏的Place- -Net Alias.. 命令可放置
网络标号。放置总线时的须注意:
●总线的名字不能是以数字结尾的;
●符号[ ]前后不能有空格;
●命名必须是名字加[ ]，如BUS[0:15]， BUS[0-15], BUS[0… 15]注意要有三点，三种中的一种;
4、信号线与总线的连接采用的是总线入口的方式，通过点击键盘的E键或者点击菜单栏中的Place- -Bug 命令。

添加差分属性

1、鼠标左键单击选中一页原理图，然后执行菜单命令“Tools-Create Differential Pair…”
2、在弹出的Create Differential Pair窗口中查找需要设置的网络名并添加到Selections中;
3、在①箭头处输入需要查找的大概网络名;
4、在②箭头处选择具体对应的网络名;
5、点击③箭头处的符号即可添加到Selections中;
6、点击④箭头处的Create即可自动创建好差分类。

DRC检查

PCB / Design Rules Check

  1. Check single node nets: 检查设计中的单端网络;
  2. Check no drivingsource and pintype…::检查器件属性无源管脚与有源管脚的连接是否正
  3. Check duplicate net name :检查重复的网络名称;
  4. Check off- page connector connect:检:测分页图纸间接口的连接性;
  5. Check hierarchical port connection: 检测阶层端口的连接性;
  6. Check unconnected bus net: 检测没有连接的总线网络;
  7. Check unconnected pins: 检查没有连接的管脚;
  8. Check power pin visible: 检测电源属性的管脚是否显示了管脚名称与编号;
  9. Check missing/illegal PCB Footprint: 检测封装名称是否填写。是否含有非法字符;
  10. CheckNormalconvertviewsystem:检测—个部件的正常视图上的pin号是否与转换视图上的
  11. Check incorrect pin group assi gnment: 检查同一个Pin Group的所有插脚是否有相同的类型
  12. Check high speed props syntax: 在设计中检查网络高速属性的语法是否正确;
  13. Check missing pin number: 检测缺失管脚编号的管脚;
  14. Checkdevicewithzeropins::检测设计中是否含有零宽度的管脚;
  15. Checkpowergroundshort:检测一个Part内部的电源网络名称是否有两个不同的网络标号名
  16. Check Name Propconsistency:检测层次原理结构中是否含有相同的网络名称;
  17. Report all net name: 报告所有网络的名称;
  18. Report off- grid object: 报告不在格点上是器件;
  19. Report hierarchical ports and off-page connection: 报告阶层端口和分页图纸间接口的连
  20. Report Misleading Tap Connection: 报告易误解的连接。
  21. Report Visible unconnected powerpins:报告所有具有可见电源属性的管脚是否有连接;
  22. Report unused part packages: 报告原理图中未使用的器件part;
  23. Report invalid Refdes:报告原理图中无效的器件位号;
  24. Report identical part references: 报告原理图相同的位号。

BOM清单输出

1、选中根目录以后，执行菜单命令Tools-Bill .命 令，来进行BOM清单的输出;
2、在Header以及Combined Property String栏中，分别列出了需要输出的元素，依次是器件的数量、器件的位号、器件的属性值;
3、缺失器件的封装属性值，需要单独加上，格式就是跟Header以及Combined Property String栏致，勾选上==Open in Excel==, 这样输出BOM清单就用EXCEL表格打开了，进行编辑整理，得到需要的BOM清单。

==Header下: … \ tPCB Footprint==

==Combined 下: …\t{PCB Footprint}==

最后改为输出==*.xsl==

原理图输出各类网表

Capture输出 Tool / Create Netlist…

  1. 第一方网表 allegro

    会创建allegro文件夹，包含五个文件

    • netlist.log 配置文件 通过这个来对错误分析
      • pstchip.dat 网表
      • pstnet.dat 网表
      • pstxprt.dat 网表
    • pxlBA.txt 配置文件

  1. 第三方网表 orTelesis64.dll

    由于其中大量使用小数点表示 所以要PCB Footprint 去替换 Part Value

    • Pads网表 orpads2k64.dll 可以不替代，且输出为 fxq.asc

    • AD网表 orprotel264.dll 可以不替代，且输出为 fxq.net
