串口方法

<1>基本内容

缩写

  • DMA：STM32中的直接内存访问控制器
  • UART：通用异步收发器
  • USART：通用同步异步收发器
  • TX：发射
  • RX：接收
  • HT：半传输完成 DMA 事件/标志
  • TC：传输完成 DMA 事件/标志
  • RTO：接收器超时 UART 事件/标志
  • IRQ：中断

模式划分

  • 轮询模式（无 DMA，无 IRQ）
    • P：应用程序正在轮询状态位以检查是否已发送/接收任何字符，并以足够快的速度读取它，以免错过任何字节
    • P：易于实现，只需几行代码
    • C：如果 CPU 不能足够快地读取寄存器，在复杂的应用程序中很容易丢失接收到的数据
    • C：仅适用于低波特率9600或更低
  • 中断模式（无 DMA）
    • P：UART触发中断，CPU跳转到服务程序分别处理每个接收到的字节
    • P：嵌入式应用中常用的方法
    • P：适用于常见的波特率115200，最高~921600波特率
    • C：对每个接收到的字符执行中断服务程序
    • C：如果为高速波特率的每个字符触发中断，可能会降低系统性能
  • DMA模式
    • DMA 用于在硬件级别将数据从 USART RX 数据寄存器传输到用户存储器。此时不需要应用程序交互，除非必要时由应用程序处理接收到的数据
    • P：从 USART 外设到内存的传输是在硬件级别完成的，无需 CPU 交互
    • P：可以很容易地与操作系统一起工作
    • P：针对最高波特率> 1Mbps和低功耗应用进行了优化
    • P：在大量数据突发的情况下，增加数据缓冲区大小可以改善功能
    • C: DMA 硬件必须提前知道要传输的字节数
    • C：如果通信失败，DMA 可能不会通知应用程序所有传输的字节

DMA模式

STM32中的DMA可以在normalorcircular模式下配置。对于每种模式，DMA需要在触发其事件（半传输完成、传输完成）之前传输多个元素。

  • 正常模式

    • DMA 以数据传输开始，一旦传输所有元素，它就会停止并将启用位设置为
    • 应用程序在传输数据时使用此模式

  • 循环模式

    • DMA从传输开始，一旦传输完所有元素（写入相应的长度寄存器），它从内存的开头开始传输更多
    • 应用程序在接收数据时使用此模式

当传输处于活动状态时，2可能会触发（除其他外）中断：

  • Half-Transfer completeHT：当 DMA 传输一半元素时触发
  • Transfer-CompleteTC：当 DMA 传输所有元素时触发

当 DMA 在循环模式下运行时，这些中断会定期触发

DMA 硬件传输的元素数量必须在传输开始前写入相关的 DMA 寄存器

优先级

Cortex-M 中有优先级类型：

  • 抢占优先级：具有较高逻辑优先级的中断可以抢占已经运行的较低优先级中断
  • 子优先级：当（或更多）中断线同时激活时，具有更高子优先级（但抢占优先级相同）的中断将首先执行；这种中断也永远不会停止CPU当前执行的中断（如果有的话）。

应用程序必须确保 DMA 和 UART 中断使用相同的抢占优先级。这是保证处理功能永远不会被自己抢占的唯一配置（DMA 中断到抢占 UART，或相反），否则最后已知的读取位置可能会损坏，应用程序将使用错误的数据运行。

检测标志

传输检测标志：

  • 接收缓冲区已满
  • 发送缓冲区为空
  • 传输结束标志

奇偶校验控制：

  • 发送奇偶校验位
  • 检查接收的数据字节的奇偶性

四个错误检测标志：

  • 溢出错误
  • 噪声检测
  • 帧错误
  • 奇偶校验错误

十个具有标志位的中断源：

  • CTS 变化
  • LIN 停止符号检测
  • 发送数据寄存器为空
  • 发送完成
  • 接收数据寄存器已满
  • 接收到线路空闲
  • 溢出错误
  • 帧错误
  • 噪声错误
  • 奇偶校验错误

主要特征

单周期乘法运算

  1. 一个周期内完成3232位乘法，或两个 16 16 位的乘法累加运算 （同样32位需要四个周期完成）
  2. 快速的中断响应
  3. 可在任何内存位置进行单周期读、修改、写操作
  4. 可采用C / C++软件编程，效率非常高

处理器主要资源

  1. 32位浮点DSP，主频 150MHZ，方便电机控制，电力设备控制及工业控制
  2. 片上存储器： FLASH—256K16位， SRAM—36K 16位； BOOT ROM—8K 16位； OPT ROM—2K 16位。 其中FLASH，OPT ROM受口令保护，可以保护用户程序
  3. 片上外设： PWM — 18路； HRPWM — 6路； CAP — 6 路； QEP — 2通道；ADC — 2 * 8 通道， 12位，80ns转换时间，0-3V输入量程；SCI — 3通道； MCBPS — 2通道； CAN—2通道； SPI — 1通道； I2C—1通道； 外部存储器扩展接口XINIF；通用输入/输出IO—88个；看门狗电路；

引脚及其功能分布

  1. JTAG引脚
  2. 时钟引脚
  3. 复位引脚
  4. 电源引脚
  5. ADC信号引脚
  6. GPIO和外设信号引脚

如何学好

  1. 基本外设： GPIO输入输出，外部中断，定时器，串口
  2. 掌握基础的模拟电路和数字电路
  3. 深入学习： SPI、IIC、SCI、ECAN、EPWM、ADC、 MCBSP、 ECAP、EQEP等
  4. 高级功能： UCOS、 FATFS、 MATALB
  5. C语言能力，多浏览论坛

最小系统组成

  1. 电源电路
  2. 复位电路
  3. 晶振电路
  4. 下载电路
